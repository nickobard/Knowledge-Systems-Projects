## HOW-TO

To run in your shell:  `python3 predikaty.py rules.tx`

### Implementation details

The implementation tries to implement forward chaining and unification from the AIMA book.

Algorithm is based on modification of the following algorithm from the book:

![forward_chaining.png](images/forward_chaining.png)

In the algorithm on the image it only gives values that satisfy certain sentence, while in our case we have to continue to gather new facts until there are no new facts possible.


The unification algorithm is straightforward because of the task simplification. 

The complete unification algorithm looks like:

![unify.png](images/unify.png)

Implementation is simplification of this algorithm.

## The Assignment
Implement your chosen variant of the forward chaining algorithm in any procedural programming language of your choice (e.g., C++, Java, Python, etc.).

The goal of the completed software is to derive all facts that can be inferred from the given knowledge base and extend the original fact base with these newly derived facts (knowledge base = fact base + rule base).

Test your implementation using a task generated by the testing server. Once your solution has been debugged, submit it through the same server.

During development, you may generate any number of test tasks. The only requirement is that each generated task must be solved and submitted back to the server within the specified time limit.

Templates are available in C++ and Python, both with a parser implementation that you may (but are not required to) use.

Since this is an educational assignment, the following simplifications apply:

1. For the purposes of this assignment, a rule is an implication whose left-hand side consists of either a single expression or a conjunction of expressions formed by predicates and the operators `=`, `\=` (not equal), or `not =` (not equal). Rules may contain only predicates, the operators `=`, `\=`, and `not =`, variables, and constants (i.e., functions are not considered in this assignment). Predicate nesting is also not considered. Furthermore, assume that rules may contain only constants that already occur as arguments in at least one fact.

2. Facts are fully instantiated atoms (i.e., they contain no variables).

3. Identifiers representing variables and constants may begin with an uppercase or lowercase letter, a digit, or an underscore. Identifiers are case-sensitive. (Therefore, the syntax differs from that of Prolog.) The syntax is formally described by the grammar (see below), although you do not need it if you use the provided parser.

4. Assume that all facts and rules are syntactically correct. Therefore, your implementation does not need to validate the syntax of the knowledge base. The provided parser already performs syntax validation. If you implement your own parser, you may rely on this assumption.

5. A template is provided with a parser implementation that converts facts and rules into a representation suitable for further processing (an object-based structure).

6. The assignment is graded out of 10 points. The efficiency or code quality of your implementation is not evaluated. Only correctness and the ability of the program to solve the task within 3 minutes are assessed.

7. *The solution must implement a forward chaining algorithm.*

You may implement any strategy for selecting a rule from the set of applicable rules. The choice of strategy is entirely up to you. Likewise, you are free to choose how your algorithm performs variable unification (substitution).
